# Project: SkyFoodV2-Vision-OAK Perception Module
- About SkyFood: https://skyfood.alphacode.com.br/
- Documentation for OAK apps (VERY IMPORTANT!): https://docs.luxonis.com/software-v3

## Project Mission & Scope
Mission: To develop a standalone, high-performance perception application for the SkyFoodV2 robot. This application will run entirely on the OAK-D camera, offloading critical vision processing from the main compute unit (NVIDIA Jetson Orin Nano). The module will provide the rest of the robot's software stack with rich, real-time data about the environment, enabling robust navigation and human-robot interaction.
Core Features (Scope):
High-Quality Depth Stream: Publish a synchronized and aligned depth map for obstacle avoidance and 3D perception.
General Obstacle Detection: Identify and locate common dynamic and static obstacles, with a primary focus on humans.
Key Landmark Detection: Identify and locate critical navigational and interactive landmarks, specifically faces and the charging dock.
ROS 2 Integration: Publish all processed data onto designated ROS 2 topics for seamless integration with the navigation (Nav2) and behavior stacks.

## System Architecture & Data Flow
The OAK-D application will be a single, cohesive pipeline. The data will flow as follows:
Input: Raw image data from the onboard color and stereo cameras.
On-Device Processing (OAK-D):
Stereo Depth: Calculate a dense depth map.
Image Manipulation: Resize and format images for multiple neural network models.
Parallel Inference:
Model 1 (General): A YOLO-based model detects humans.
Model 2 (Faces): A lightweight model (e.g., YuNet) detects faces.
Model 3 (Dock): A custom-trained model detects the charging dock.
Spatial Calculation: Combine NN bounding boxes with the depth map to determine the real-world 3D coordinates (X, Y, Z) of all detected objects.
Output (to Jetson): The processed data is sent from the OAK-D to the Jetson via Ethernet (both are connected to a PoE switch. The OAK-D is already setup with a static IP address).
ROS 2 Publisher: A ROS 2 node on the Jetson receives the data from the OAK-D and publishes it onto the appropriate ROS 2 topics.


This project is divided into three main phases. Each phase is a prerequisite for the next.
